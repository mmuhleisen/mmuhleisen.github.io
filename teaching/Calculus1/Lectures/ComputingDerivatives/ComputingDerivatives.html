Title

Computing Derivatives

Exposition

One of the major goals in a calculus class is to answer the following question.

Question

1

How do you calculate the derivative of an elementary function?

Exposition

Recall that a function f:U→ℝ is called elementary if it is one of the building blocks listed in this example , or if it is a combination of building blocks formed using the four arithmetic operations, restriction, and composition.

Let’s start by answering the question for the building blocks. This amounts to giving a large table of derivatives you can find on the inside covers of many calculus textbooks.

Example

2

If an elementary function given by one of the expressions in the first column is defined on an open interval U=(a,b), then the derivative is defined on the same interval U and given by the expression in the second column.

Exposition

This table is a good start to answering the question, but we also need to know how to find the derivatives of combinations of building blocks. To that end, one uses the following derivative rules.

Theorem

3

Let U=(a,b) be an interval. Then the following hold.

  1. (Linearity) If f:U→ℝ and g:U→ℝ are two elementary functions, then the derivative of f+g is f′+g′. This is expressed in the equation

    dd⁢x⁢(f⁢(x)+g⁢(x))=d⁢fd⁢x+d⁢gd⁢x
    and equivalently in the string diagram 

			
			stateDiagram
			direction LR
			state SumOfFunctions {
				beforeOne: f
				beforeTwo: g
				beforeSum: +
				[*] --> beforeOne
				[*] --> beforeTwo
				beforeOne --> beforeSum
				beforeTwo --> beforeSum
				beforeSum --> [*]
			}
			state DerivativeOfSum {
				afterOne: f’
				afterTwo: g’
				afterSum: +
				[*] --> afterOne
				[*] --> afterTwo
				afterOne --> afterSum
				afterTwo --> afterSum
				afterSum --> [*]
			}
			SumOfFunctions --> DerivativeOfSum

  1. (Leibniz Rule) If f:U→ℝ and g:U→ℝ are two functions, then the derivative of f⁢g is f′⁢g+f⁢g′. This is expressed in the equation

    dd⁢x⁢(f⁢(x)⋅g⁢(x))=d⁢fd⁢x⋅g+f⋅d⁢gd⁢x
    and equivalently in the string diagram 

			
			stateDiagram
			direction LR
			state ProductOfFunctions {
				beforeOne: f
				beforeTwo: g
				beforeProduct: •
				[*] --> beforeOne
				[*] --> beforeTwo
				beforeOne --> beforeProduct
				beforeTwo --> beforeProduct
				beforeProduct --> [*]
			}
			state DerivativeOfProduct {
				afterOne: f
				afterOnePrime: f’
				afterTwo: g
				afterTwoPrime: g’
				afterSum: +
				afterProductOne: •
				afterProductTwo: •
				[*] --> afterOne
				[*] --> afterOnePrime
				[*] --> afterTwo
				[*] --> afterTwoPrime
				afterOnePrime --> afterProductOne
				afterTwo --> afterProductOne
				afterOne --> afterProductTwo
				afterTwoPrime --> afterProductTwo
				afterProductOne --> afterSum
				afterProductTwo --> afterSum
				afterSum --> [*]
			}
			ProductOfFunctions --> DerivativeOfProduct

  1. (Chain Rule) Let V=(c,d) be another interval. If f:U→V and g:V→ℝ are two elementary functions, then the derivative of g∘f is (g′∘f)⁢f′. This is expressed in the equation

    dd⁢x(g(f(x))=d⁢gd⁢f⋅d⁢fd⁢x.
    or in the string diagram 

			
			stateDiagram
			direction LR
			state CompositeOfFunctions {
				beforeOne: f
				beforeTwo: g
				[*] --> beforeOne
				beforeOne --> beforeTwo
				beforeTwo --> [*]
			}
			state DerivativeOfComposite {
				afterOne: f
				afterOnePrime: f’
				afterTwoPrime: g’
				afterProduct: •
				[*] --> afterOne
				[*] --> afterOnePrime
				afterOne --> afterTwoPrime
				afterTwoPrime --> afterProduct
				afterOnePrime --> afterProduct
				afterProduct --> [*]
			}
			CompositeOfFunctions --> DerivativeOfComposite

Exposition

Now we know what the derivatives of the building blocks are, and we also know what the derivatives of a sum, a product, and a composite of functions are. Therefore we can answer the original question for an arbitrary elementary function by deconstructing it into sums, products, and composites of building blocks. To put it differently, derivatives of elementary functions are computed recursively.

We will do many, many examples in class at this point.
